# WealthFlow AI

A full-stack personal finance management web application with AI-powered receipt scanning, budget alerts, and financial insights.

**Live Demo:** [wealthflow-ai.replit.app](https://wealthflow-ai.replit.app)

---

## Features

- **JWT Authentication** — Secure registration and login with bcrypt password hashing and 7-day signed tokens
- **Account Management** — Create multiple accounts with individual balance tracking and monthly budget limits
- **Transaction Tracking** — Add income/expense transactions with category tagging, pagination, and filters
- **AI Receipt Scanner** — Upload a receipt photo and Gemini Vision API auto-fills the transaction form (amount, merchant, category, date)
- **Budget Alerts** — Automated email alerts at 80% and 100% budget usage with deduplication via a `ScheduledEmail` table
- **Recurring Transactions** — Set daily/weekly/monthly/yearly recurring transactions processed by a background cron job
- **Monthly Reports** — Auto-generated financial reports with income vs expense breakdown and category-wise spending
- **AI Financial Tips** — Personalized financial tips generated by Gemini AI based on spending patterns
- **Redis Caching** — Cache-aside pattern with 10-minute TTL for dashboard summaries, reducing DB load
- **Rate Limiting** — Arcjet fixed-window rate limiter (60 req/min) and shield protection on all API routes

## Architecture

```
client/                  # React frontend (Vite)
  src/
    context/             # AuthContext — global auth state
    pages/               # Dashboard, Accounts, Transactions, Reports
    components/          # Layout, UI components
    utils/               # Axios instance with JWT interceptor

server/                  # Express backend
  controllers/           # Business logic (user, account, transaction, receipt)
  middlewares/           # JWT auth guard, Arcjet rate limiter
  routes/                # Route definitions
  services/              # Gemini AI, Cron jobs (recurring txns + monthly reports)
  utils/                 # Redis client, Prisma client, Email sender, Budget alert
  prisma/                # Database schema and migrations
```

## Key Design Decisions

**ACID Transactions** — Balance updates and transaction record creation happen inside a Prisma `$transaction` block. If either step fails, both are rolled back — no partial state.

**Cache-Aside** — Redis is checked before every DB read for dashboard data. On writes, the relevant cache keys are invalidated. Falls back gracefully if Redis is unavailable.

**Non-blocking Budget Alerts** — `checkAndSendBudgetAlert()` is called without `await` after a transaction is saved, so the API response is not delayed by email sending.

**Gemini Model Fallback** — Receipt parsing tries 3 models in sequence (`gemini-2.0-flash-lite` → `gemini-2.0-flash` → `gemini-1.5-flash-8b`). If all fail due to quota limits, demo data is returned so the app never crashes during a demo.

**SPA Routing** — Express serves the built React bundle and returns `index.html` for all non-API routes, enabling client-side navigation without a separate static file server.

## Local Development

### Prerequisites
- Node.js 20+
- PostgreSQL database (or Neon free tier)
- Redis instance (or Upstash free tier)
- Google Gemini API key
- Gmail account with App Password

### Setup

```bash
# Clone the repo
git clone https://github.com/Arjun55666/wealthflow-ai
cd wealthflow-ai

# Install server dependencies
npm install --prefix server

# Install client dependencies
npm install --prefix client

# Create server/.env
cp server/.env.example server/.env
# Fill in: DATABASE_URL, REDIS_URL, JWT_SECRET, GEMINI_API_KEY, EMAIL_USER, EMAIL_PASS

# Run database migrations
cd server && npx prisma migrate deploy

# Start development servers
cd server && node app.js &
cd client && npm run dev
```

The frontend runs on `http://localhost:5173` and the backend on `http://localhost:3000`.

## API Endpoints

### Auth
| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/auth/register` | Register a new user |
| POST | `/api/auth/login` | Login and receive JWT |
| GET | `/api/auth/me` | Get current user (protected) |

### Accounts
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/accounts` | Get all accounts |
| POST | `/api/accounts` | Create account |
| PUT | `/api/accounts/:id` | Update account |
| DELETE | `/api/accounts/:id` | Delete account |

### Transactions
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/transactions` | Get transactions (paginated + filtered) |
| POST | `/api/transactions` | Create transaction |
| DELETE | `/api/transactions/:id` | Delete transaction |

### AI & Reports
| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/receipts/scan` | Scan receipt image with Gemini Vision |
| GET | `/api/graph/summary` | Monthly income/expense summary |
| GET | `/api/graph/categories` | Expense breakdown by category |
| GET | `/api/graph/monthly` | Monthly chart data (Redis cached) |
| GET | `/api/graph/tip` | AI-generated financial tip |
